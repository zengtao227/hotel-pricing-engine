# 建模路线

## 1. 核心思路

酒店动态定价不应该被建成“预测历史房价”的项目。历史房价只是人工策略的结果，直接学习它会复制过去的定价习惯，未必能提升收益。

更合理的路线是：

1. 预测需求
2. 估计价格对需求的影响
3. 对候选价格做收益模拟
4. 输出风险调整后的推荐价

当前程序已经实现第一版候选价收益模拟和价格弹性基线。完整数学说明见 [revenue-optimization-model.md](revenue-optimization-model.md)。

## 2. 基线模型

第一版建议先建立可解释基线。

### 2.1 需求预测

预测目标：

- 某入住日期和房型的最终售出间夜数
- 某观察日期之后还会新增多少预订

候选方法：

- 历史同期平均
- 按星期、月份、节假日分组的规则模型
- 线性回归或树模型
- 时间序列基线

输入特征：

- 入住月份
- 星期几
- 是否周末
- 是否节假日
- 提前期
- 当前已订房量
- 剩余库存
- 最近 7 天或 14 天 pickup
- 历史同期入住率
- 淡旺季需求倍率（由酒店配置中心的季节日历提供）
- Pickup 速度比率（当前 14 天 pickup ÷ 历史同类日期中位数）

### 2.2 价格响应估计

目标是估计不同价格下的预计成交量，而不是直接预测价格。

可先使用简化方法：

- 按价格区间观察历史成交率
- 按房型和季节估计价格弹性
- 使用规则限制极端价格变化

后续可尝试：

- 转化概率模型
- 因果推断或 uplift 思路
- 分层贝叶斯价格弹性模型
- 强化学习或 bandit，但必须在有足够实验数据后再考虑

### 2.3 收益优化

对每个候选价格计算：

```text
已锁定收入 = 观察日已经成交订单的房费收入
未来需求 = demand_model(features, candidate_price)
预计新增售出 = min(未来需求, 剩余库存)
expected_revenue = 已锁定收入 + candidate_price * 预计新增售出
```

推荐价不一定是数学上最高的点，还要考虑：

- 预测置信度
- 价格变动幅度
- 品牌定位
- 人工设定的最低价和最高价
- 渠道价格一致性
- 取消风险

当前 v1 实现使用常数价格弹性模型：

```text
FutureDemand(p) = max(D0 - S0, 0) * (p / p0) ^ epsilon
ExpectedRevenue(p) = R0 + p * min(C, FutureDemand(p))
```

其中 `S0` 是观察日已售间夜，`R0` 是已锁定收入，`C` 是剩余库存，`epsilon` 是价格弹性。系统会在单次调价幅度、房型最低价/最高价和价格尾数规则内枚举候选价，并选择预期收益最高的可执行价格。若收益提升低于最小阈值，则建议保持当前价。

## 3. 推荐输出

每条推荐至少包含：

- `stay_date`
- `room_type`
- `current_price`
- `recommended_price`
- `action`: increase、decrease、hold
- `expected_revenue_delta`
- `current_expected_revenue`
- `recommended_expected_revenue`
- `demand_forecast_at_current_price`
- `expected_sold_rooms`
- `demand_elasticity`
- `confidence`
- `main_reasons`
- `risk_flags`

示例解释：

```text
推荐上调：未来 14 天 pickup 高于历史同期，剩余库存低于正常水平，
且该日期为周末。建议从 980 调整到 1080，预计 RevPAR 提升 6%。
```

## 4. 回测方法

回测需要回答：

- 如果当时使用推荐价，预期收益是否高于实际历史策略
- 模型是否在旺季、淡季、周末和临近入住日期都表现稳定
- 推荐的调价幅度是否过于激进
- 数据不足时是否能正确降低置信度

基础回测指标：

- RevPAR uplift
- ADR uplift
- Occupancy change
- 需求预测误差
- 推荐方向准确率
- 高风险建议比例

## 5. 风险控制

MVP 必须加入规则护栏：

- 单次调价幅度限制
- 最低价和最高价限制
- 低置信度只建议人工复核
- 数据缺失时不强行推荐
- 异常日期必须标记原因

## 6. 模型演进路径

### 阶段 1：规则 + 统计基线

建立指标、历史同期对比、简单需求预测和候选价格模拟。

当前状态：已实现 v1 候选价收益模拟，并已在 Backtest tab 展示价格弹性收益回测、候选价收益曲线和静态销量对照。同时已加入淡旺季日历信号和 Pickup 速度基准对比，两者均以评分方式叠加到推荐理由中，不强制修改收益模拟基线（保留系统在旺季低需求时向下调价的能力）。

### 阶段 2：机器学习需求预测

引入树模型或其他可解释模型，提高需求预测准确度。

### 阶段 3：价格弹性建模

更系统地估计不同房型、季节和提前期的价格响应。

### 阶段 4：实验和自动化

在有足够验证数据后，再考虑自动调价、A/B 测试或 bandit 策略。
