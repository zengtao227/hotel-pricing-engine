# Roadmap

## Phase 0: 需求定义

目标：明确系统要解决的问题和数据边界。

交付物：

- 项目 README
- 项目 CONTEXT
- 产品需求文档
- 数据需求文档
- 建模路线文档
- 待确认问题清单

状态：进行中

## Phase 1: 数据原型

目标：使用样例数据或真实脱敏数据跑通分析流程。

交付物：

- 样例数据格式
- 数据质量检查规则
- 每日房型级指标表
- 历史入住率、ADR、RevPAR 分析

完成标准：

- 能从 CSV 或表格生成核心指标
- 能识别缺失、重复和异常字段
- 能输出可供建模使用的聚合数据

## Phase 2: 基线定价建议

目标：实现第一版可解释定价建议。

交付物：

- 需求预测基线
- 候选价格收益模拟
- 推荐价、建议动作和解释
- 回测报告

完成标准：

- 能对未来日期输出推荐价
- 能解释推荐理由
- 能在历史数据上做策略回测
- 能标记低置信度或数据不足场景

## Phase 3: 决策看板

目标：让收益经理可以每天查看和复核建议。

交付物：

- 日期和房型级推荐列表
- 当前价与推荐价对比
- 风险提示
- 收益影响估算
- 关键指标趋势图

完成标准：

- 用户能快速定位需要调价的日期
- 每条建议都有足够解释
- 可导出人工执行用的调价清单

## Phase 4: 真实业务集成

目标：接入真实数据源和更完整业务流程。

可能交付物：

- PMS 或 OTA 数据导入
- 竞品价格采集
- 多渠道价格策略
- 人工审批记录
- 模型版本与审计日志

完成标准：

- 数据更新流程稳定
- 推荐结果可审计
- 人工调价闭环可追踪

## Phase 5: 自动化探索

目标：在充分验证后探索更自动化的调价能力。

前置条件：

- 有足够长期回测结果
- 有人工审批历史
- 有业务方认可的风险护栏
- 能快速回滚错误定价

可能方向：

- 半自动调价
- A/B 测试
- Bandit 策略
- 多酒店组合收益优化
