# Roadmap

## 当前状态概览

项目已经从最初的需求定义推进到可演示的 Streamlit MVP，并已经覆盖了原 Phase 1、Phase 2 和 Phase 3 的大部分内容。

当前已具备：

- CSV / demo 数据加载
- 每日房型级 Occupancy、ADR、RevPAR、Pickup 指标
- 可解释的规则型调价建议
- 中文、英文、德文、法文界面
- 销售演示看板
- 多语言 Excel 报表导出
- 中国酒店演示用 6/8/9 价格尾数规则
- 酒店配置：房型、基准价、最低价、最高价、周末加价
- 价格审批与发布：最终批准价、人工修改标记、模拟推送
- 审批与推送日志持久化到本地 CSV
- 回测分析：基于历史已发生日期的静态销量回测

---

## Phase 0: 需求定义

目标：明确系统要解决的问题和数据边界。

交付物：

- 项目 README
- 项目 CONTEXT
- 产品需求文档
- 数据需求文档
- 建模路线文档
- 待确认问题清单

状态：已基本完成，后续随产品演进持续更新。

---

## Phase 1: 数据原型

目标：使用样例数据或真实脱敏数据跑通分析流程。

交付物：

- 样例数据格式
- 数据质量检查规则
- 每日房型级指标表
- 历史入住率、ADR、RevPAR 分析

完成标准：

- 能从 CSV 或表格生成核心指标
- 能识别缺失、重复和异常字段
- 能输出可供建模使用的聚合数据

状态：已完成 MVP 版本。

下一步增强：

- 更强的数据质量报告
- 字段自动识别与映射
- 房型名称自动匹配
- 上传真实酒店 Excel 的容错处理

---

## Phase 2: 基线定价建议

目标：实现第一版可解释定价建议。

交付物：

- 需求信号基线
- 推荐价、建议动作和解释
- 价格尾数规则
- 价格上下限保护
- 回测报告

完成标准：

- 能对未来日期输出推荐价
- 能解释推荐理由
- 能在历史数据上做策略回测
- 能标记低置信度或数据不足场景

状态：已完成可演示版本。

当前回测说明：

- 当前实现为静态销量回测，即假设售出房间数不随价格变化。
- 适合验证推荐方向和解释逻辑。
- 后续需要升级为带价格弹性和需求预测的收益模拟。

下一步增强：

- 候选价格集合收益模拟
- 价格弹性模型
- 取消风险调整
- 节假日和活动日因子

---

## Phase 3: 决策看板

目标：让收益经理可以每天查看和复核建议。

交付物：

- 日期和房型级推荐列表
- 当前价与推荐价对比
- 风险提示
- 收益影响估算
- 关键指标趋势图
- 多语言界面
- Excel 报表导出

完成标准：

- 用户能快速定位需要调价的日期
- 每条建议都有足够解释
- 可导出人工执行用的调价清单

状态：已完成演示版本。

下一步增强：

- 更清晰的客户演示首页
- 产品卖点卡片
- 按房型/日期/动作的更强筛选
- 更美观的 Excel 报表样式

---

## Phase 4: 价格审批与发布

目标：把系统从“建议工具”升级为“人工复核后可发布的运营流程”。

已完成：

- 最终批准价
- 人工修改自动标记
- 批量采纳推荐价
- 已推送/人工修改/已拒绝颜色提示
- 一键模拟推送
- 审批与推送日志下载
- 审批与推送日志持久化到 `data/audit_logs/price_approval_publishing_log.csv`

下一步增强：

- 审批日志改为 SQLite 或 Postgres
- 记录登录用户和角色
- 增加审批前确认弹窗
- 支持撤销/回滚一次推送
- 支持导出 Channel Manager 上传模板

---

## Phase 5: 真实业务集成

目标：接入真实数据源和真实价格发布流程。

推荐顺序：

1. Excel / CSV 导出
2. Channel Manager 导入模板
3. 接入一个真实 PMS 或 Channel Manager API
4. 抽象多系统 adapter

可能交付物：

- PMS 或 OTA 数据导入
- 竞品价格采集
- 多渠道价格策略
- 人工审批记录
- 模型版本与审计日志

完成标准：

- 数据更新流程稳定
- 推荐结果可审计
- 人工调价闭环可追踪

---

## Phase 6: 自动化探索

目标：在充分验证后探索更自动化的调价能力。

前置条件：

- 有足够长期回测结果
- 有人工审批历史
- 有业务方认可的风险护栏
- 能快速回滚错误定价

可能方向：

- 半自动调价
- A/B 测试
- Bandit 策略
- 多酒店组合收益优化

---

## 当前最高优先级

1. 把 Streamlit 服务从 `nohup` 改成 systemd。
2. 把审批日志从 CSV 升级为 SQLite。
3. 增强回测：从静态销量回测升级为价格弹性回测。
4. 增强真实数据导入和数据质量报告。
5. 做一个 Channel Manager 导入模板导出功能。
