# 待确认问题

## 数据

- 是否已经有真实酒店历史订单数据？
- 数据是一家酒店，还是多家酒店？
- 历史数据覆盖多久，是否至少有 12 个月？
- 是否有每日库存数据？
- 是否能拿到历史价格快照，还是只有最终成交价？
- 取消订单是否有取消时间？
- 是否能区分不同渠道和价格计划？

## 业务边界

- 第一版面向单酒店，还是多酒店？
- 第一版面向所有房型，还是先选一个主力房型？
- 推荐价需要按天给出，还是按入住日期区间给出？
- 是否有最低价、最高价、单次涨跌幅等业务限制？
- 调价建议是每日生成，还是按需生成？

## 模型目标

- 第一阶段优化目标优先用 RevPAR，还是总房费收入？
- 是否需要同时约束入住率不能低于某个阈值？
- 对收益提升和入住率稳定，哪个更重要？
- 是否需要对取消率做单独建模？

## 外部数据

- 是否需要纳入竞品酒店价格？
- 是否有稳定来源获取当地活动和节假日数据？
- 是否需要考虑天气或交通数据？

## 产品形态

- 第一版是命令行报告、Notebook、网页看板，还是静态报表？
- 推荐结果是否需要导出为 Excel？
- 是否需要记录人工是否接受推荐价？
- 是否需要多用户权限？

## GitHub 项目管理

- 是否使用 issues 记录每个阶段任务？
- 是否需要建立 project board？
- 是否采用中文文档为主，还是后续补充英文 README？
