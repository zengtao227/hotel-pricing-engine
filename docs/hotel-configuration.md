# 酒店配置中心开发说明

## 1. 目标

酒店配置中心用于把原来写死在 demo 数据中的价格规则，升级为每家酒店可单独配置的业务参数。

第一版目标：

- 每家酒店可以设置基本信息。
- 每个房型可以设置基准价、最低价、最高价和周末加价。
- 系统可以选择是否用配置价覆盖当前价。
- 推荐价会受到最低价和最高价保护。
- 配置可以导出为 JSON，后续可导入复用。

这一步让产品从“演示工具”向“可定制交付工具”前进。

## 2. 当前实现

新增文件：

```text
src/hotel_config.py
```

主要功能：

- `DEFAULT_HOTEL_CONFIG`：默认酒店配置。
- `ensure_hotel_config()`：确保 Streamlit session 中有酒店配置。
- `render_hotel_configuration(...)`：渲染酒店配置页面。
- `apply_config_to_current_prices(...)`：根据配置生成当前价。
- `room_bounds_from_config(...)`：把每个房型的最低价、最高价传给推荐引擎。
- `config_to_json_bytes(...)`：导出 JSON 配置。
- `load_config_from_upload(...)`：上传并导入 JSON 配置。

## 4. 配置如何影响价格

### 4.1 当前价

如果开启：

```text
用配置价覆盖当前价
```

系统会根据：

```text
当前价 = 房型基准价 + 周末加价
```

再通过价格尾数规则处理。

例如：

```text
标准大床房基准价：388
周末加价：40
原始当前价：428
价格尾数规则：中国酒店吉祥尾数（6/8/9）
最终当前价：428
```

如果关闭该选项，系统会使用上传的 `current_prices.csv` 或数据文件中的 `current_price`。

### 4.2 推荐价

推荐引擎会先根据入住率、库存、pickup 和时间压力计算推荐价，然后应用：

```text
最低价保护
最高价保护
价格尾数规则
```

例如：

```text
系统原始推荐价：715
高级大床房最高价：688
最终推荐价：688
风险提示：已应用最高价保护
```

## 5. 当前 UI 结构

主页面现在包含五个标签页：

1. 销售演示看板
2. 调价建议
3. 价格审批与发布
4. 酒店配置
5. 数据预览

酒店配置页面包含：

- 酒店名称
- 城市
- 市场定位
- 货币
- 是否用配置价覆盖当前价
- 默认语言
- 房型价格配置表
- 配置 JSON 上传
- 配置 JSON 下载

## 6. 当前限制

当前配置仍然保存在 Streamlit session 中：

- 刷新页面或重启服务后可能回到默认配置。
- 还没有保存到数据库。
- 多用户之间暂时不共享配置。
- 上传 JSON 可以作为临时持久化方案。

## 7. 下一步开发建议

### 7.1 配置持久化

优先做：

```text
SQLite 或 JSON 文件持久化
```

后续 SaaS 化时改成：

```text
PostgreSQL
```

### 7.2 多酒店管理

后续需要支持：

- 一个账号管理多个酒店。
- 每个酒店独立房型配置。
- 每个酒店独立价格规则。
- 不同酒店之间数据隔离。

### 7.3 房型映射

真实接入 PMS / Channel Manager 时，需要增加：

- 内部房型名称
- PMS 房型 ID
- Channel Manager 房型 ID
- OTA 房型映射
- 价格计划 ID

### 7.4 规则配置增强

后续可增加：

- 节假日加价规则
- 活动日规则
- 提前预订折扣
- 临近入住折扣
- 周中 / 周末不同策略
- 长住价格规则

### 7.5 发布模板导出

下一步建议增加：

```text
发布模板 CSV 导出
```

它与普通调价建议报表不同，应该直接用于导入 Channel Manager 或 PMS。
